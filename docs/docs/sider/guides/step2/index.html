快速部署

提示

以docker compose方式为例

安装包​

Download Bind-Webadmin v1 
# wget https://bind-webadmin.com/download/bind-webadmin.v1.tar.gz
# tar zxvf bind-webadmin.v1.tar.gz

安装​

# cd bind-webadmin.v1
# sh install.sh
# tree /app
/app                     # 安装在 /app 目录。在安装前需保证本宿主中没有该目录。
├── cert
│   ├── dhparams.pem
│   ├── fullchain.pem    # 域名证书
│   └── privkey.pem      # 证书私钥
├── docker-compose.yml   # docker compose 配置文件
├── env.txt              # 主配置文件(环境变量)。供各个容器初始化时统一调用
├── mysql
├── server
│   └── app.conf         # api接口配置文件
└── start.sh             # 启动

以NS_DOMAIN='hello.com'为例

# sh start.sh

打开webadmin​

提示

由于以hello.com为例，用户需在本地hosts文件中添加如下记录，假设192.168.3.7为宿主ip.

    hello.com      A 192.168.1.7
    www.hello.com  A 192.168.1.7
    api.hello.com  A 192.168.1.7

在chrome中打开"http://hello.com",默认帐号

  • admin/admin1111mm
  • demo/demo1111mm

正常情况下出现如login页面

配置文件说明​

env.txt主配置文件​

env.txt主配置文件说明

# 提示
# 各个容器会启动时读取如下值并到更新自己的相应配置中。

# NS_DOMAIN值必须与bind_backend中的配置文件app.conf中的ns变量值相同。
NS_DOMAIN='hello.com'


# mysql基本配置
MYSQL_HOST=bind_mysql
MYSQL_ROOT_PASSWORD=mysql+888
MYSQL_ALLOW_EMPTY_PASSWORD=no
MYSQL_POST=3306  
MYSQL_DB=BindDB
MYSQL_USERNAME=root
MYSQL_USERPWD=mysql+888
# 提示
# 1.MYSQL_POST值必须与mysql容器启动时的配置相同
# 2.MYSQL_HOST值必须与mysql空器的服务名称相同
# 3.当采用root时，MYSQL_USERPWD与MYSQL_ROOT_PASSWORD相同
# 4.采用非root方式，需手工创建帐号

# mysql运行环境(初始化时中文支持)
LANG=C.UTF-8
MYSQL_CHARSET=utf8mb4
MYSQL_COLLATION=utf8mb4_unicode_ci


#后面API接口(不要以/为结尾)
#该值必须与后端接口对外开放的配置保持一致，即接口外部域名和端口
#为外部的api接口。
#API_URL=http://192.168.3.7:9090
#API_URL_SSL=http://192.168.3.7:9090

#若采用本系统的api接口，此值为
# API_URL=http://api.${NS_DOMAIN}
# 若有证书，需配置为
# API_URLs=https://api.${NS_DOMAIN}
#API_URL=http://192.168.3.7:9090
SSL=false
API_URL=http://api.${NS_DOMAIN}
API_URL_SSL=https://api.${NS_DOMAIN}


#----------------------------
#前端参数
ICP=粤ICP备9999999999号(示例)
OTHERCODE=粤公网安备88888888888888号(示例)
LOGIN_FOOTER_TEXT='Mobile: 13751090806    email: guofs@139.com'

#title配置
APP_TITLE=mmDNS724  

#sider页脚
Footter_line_sider_1=test88
Footter_line_sider_2='mm-dns.com'

#普通页面页脚
Footter_line_1='DNS自主系统'
Footter_line_2='by mm-dns.com'

#前端在产生api Token时为该token配置的TTL
#1年=1000*60*60*24*365*1=31536000000
TTLApiToken=315360000000

#前端所有cookie的TTL值
#1天:60*60*1000*24
TTLCookie=86400000

其中NS_DOMAIN参数是bind-webadmin主域名服务器的域名。

提示

  • 系统搭建者用户只需修改NS_DOMAIN参数,其它的可以不用修改。本例以NS_DOMAIN='hello.com'为例
  • 若暂时没有SSL证书，需配置SSL=false

api后台配置文件app.conf​

#httpaddr = "127.0.0.1"
#此端口名称不能更改，已写入相关nginx配置中
httpport=9090
#运行模式：dev、test、prod
runmode=dev
#testoption=abc
appname=beego

[dev]
ver="v0.1.6"
# 创建容器时，如下6个参数务必使用如下初始化值，即将该文件提供给容器时，需采用如下值。
mysql_host=bind_mysql
mysql_port=3306
mysql_dbname=BindDB_test
mysql_user=root_root
mysql_password=mysql88899
ns=testabcxyz123.org
# 如上述初始化值不可更改，已写入docker compose初始化脚本中。

# 数组配置方式
# 域名类型
domaintypes=".com .net .org .cn .io .top .com.cn .tech .info"
# 解析记录，暂不支持ns解析
recordtypes="A MX CNAME TXT AAAA"   
# TTL(必须有按照如下格式书写)
ttl="1 1sec,5 5sec,30 30sec,60 1min,300 5min,1800 30min,3600 1h,43200 12h,86400 24h"

#必须是16、24、32位字节的字串。此值必须提供给解密端
#此值用于解析api的授权token的加解密。
#前端生成apiToken加密,后端存储并解密。
#KeyToken="abc123oweruwierw11111111"
KeyToken="6s8MQjzqRfMR6ej80a8AuBjn"

#页面认证key。此值由后端提供。
#必须是16、24、32位字节的字串。此值必须提供给解密端
#react解密使用，由后端提供。前后端务必保持一致。
KeyAuth="JaWAhEzhzdYdqVUuUhRNK0o0"

#后端使用
#用户认证时，服务端创建的token时的TTL

# 1小时: 60
