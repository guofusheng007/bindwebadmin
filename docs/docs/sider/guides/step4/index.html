记录解析​

解析配置​

解析配置，由后台app.conf配置文件中的如下参数决定，管理员可根据需要自由配置。

  • domaintypes,域名类型。
    domaintypes=".com .net .org .cn .io .top .com.cn .tech .info"
  • recordtypes，记录类型，不支持NS记录。
    recordtypes="A MX CNAME TXT AAAA"
  • ttl，记录在用户端cache中的生效时间,如下格式,最小1秒。
    ttl="1 1sec,5 5sec,30 30sec,60 1min,300 5min,1800 30min,3600 1h,43200 12h,86400 24h"
...

# 数组配置方式
# 域名类型
domaintypes=".com .net .org .cn .io .top .com.cn .tech .info"

# 解析记录，暂不支持ns解析
recordtypes="A MX CNAME TXT AAAA"   

# TTL(必须有按照如下格式书写)
ttl="1 1sec,5 5sec,30 30sec,60 1min,300 5min,1800 30min,3600 1h,43200 12h,86400 24h"

API解析​

bind-webadmin提供3种办法。

json字串方式​

此方式可解析任何类型的记录。

curl -i -XPOST "https://mm-dns.com/updatejson" -d '{
  "data":"9.9.9.9",
  "token":"5HXU...qaQAFDcnhezso"
}
'

其中

  • data为解析记录的解析值，此时可以处理任何解析类型的解析值。
  • token为解析记录的toke值

例如

# curl -XPOST "https://mm-dns.com/updatejson" -d '{
  "data":"9.9.9.9",
  "token":"5HXU...qaQAFDcnhezso"
}
'
# 返回
{"info":"ok"}

解析效果检验

C:\Users\Administrator>nslookup -qt=a oa.daatu.com 8.8.8.8
服务器:  dns.google
Address:  8.8.8.8

非权威应答:
名称:    oa.daatu.com
Address:  9.9.9.9

http Header方式​

http方式，只适应于A记录，自动查找公网IP并更新。用户可写排程更新。

# curl \
-H "token:5HXU...qaQAFDcnhezso" \
-XGET "https://mm-dns.com/updateheader"

# 返回
{"info":"ok"}

测试结果

C:\Users\Administrator>nslookup -qt=a oa.daatu.com 8.8.8.8
服务器:  dns.google
Address:  8.8.8.8

非权威应答:
名称:    oa.daatu.com
Address:  119.136.197.11

客户端方式​

Download Bind-Webadmin client Linux and windows

每5秒自动识别用户当前ip是否有变化，若有变化，就更新记录(公网IP或用户本地IP)。

配置文件名称：conf.json. 配置文件与更新程序在同一目录

# cat conf.json
{
    "public":true,
    "token":"5HXU...qaQAFDcnhezso",
    "rate": 2,
    "srhost":"https://mm-dns.com",
    "getip":"https://mm-dns.com/ip"
}

参数说明

  • public,是否更新公网ip或是用户本机本地ip
  • token,解析记录的授权token
  • rate,心跳周期，表示多少秒重新验测一下网络。默认2秒测一次。
  • srhost,更新解析的DNS服务器
  • getip,获取本地公网ip的api,可采用第三方(如https://ipinfo.io/ip)或自建服务。

mm-dns.com提供如下两种格式返回用户当前ip
